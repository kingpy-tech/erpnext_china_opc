# ERPNext 工作流与审批配置实战

ERPNext 内置了完整的工作流引擎，支持多级审批、条件路由和自动通知。本文以实际业务场景为主线，带你从零搭建一套可落地的审批体系。

---

## 一、工作流基础：状态机与转换条件

ERPNext 的工作流本质是一个**有限状态机**，每个单据在任意时刻处于某一"状态"，通过"动作"触发"转换"跳到下一状态。

核心概念：

- **状态（State）**：单据当前所处阶段，如"草稿 → 待审批 → 已批准 → 已拒绝"
- **转换（Transition）**：从一个状态到另一个状态的规则，包含触发条件和允许角色
- **动作（Action）**：用户在界面上看到的按钮，如"提交审批"、"批准"、"拒绝"

**配置入口**：`设置 → 工作流 → 新建工作流`

关键字段说明：

| 字段 | 说明 |
|------|------|
| 文档类型 | 绑定到哪种单据，如"费用申请" |
| 工作流状态字段 | 存储当前状态的字段名（需提前在单据中创建） |
| 发送邮件提醒 | 状态变更时自动触发邮件 |
| 是否激活 | 同一文档类型只能有一个激活的工作流 |

> **截图说明**：在工作流编辑页面，"状态"标签页列出所有状态节点，"转换规则"标签页定义每条跳转路径及其允许角色。

## 二、审批流配置

### 2.1 多级审批

适用于需要逐级上报的场景（如费用报销：员工 → 部门经理 → 财务总监）。

配置要点：
1. 为每个审批层级创建独立状态，如 `Pending L1`、`Pending L2`
2. 每条转换规则绑定对应角色（Role），只有该角色成员才能触发动作
3. 在"允许编辑"列中控制各状态下哪些字段可修改

### 2.2 并行审批

ERPNext 原生不支持真正的并行分支，但可通过以下方式模拟：

- 使用**分配任务（Assignment）**同时指派多人，配合自定义脚本检查所有人是否已操作
- 或借助 **Frappe Workflow Actions** 插件实现 AND/OR 条件聚合

### 2.3 条件审批

在转换规则中填写"条件"字段（Python 表达式），实现动态路由：

```python
# 示例：金额超过 5000 元时跳转到总经理审批
doc.total_amount > 5000
```

```python
# 示例：仅限本部门提交
doc.department == frappe.get_value("Employee", frappe.session.user, "department")
```

> **截图说明**：转换规则编辑行中，"条件"列支持直接输入 Python 表达式，`doc` 指当前单据对象。

## 三、常见业务场景

### 3.1 采购审批

**流程**：草稿 → 部门经理审批 → 采购经理审批 → 已批准

- 绑定文档类型：`Purchase Order`
- 建议在"已批准"状态将单据设为只读，防止事后篡改
- 可在转换条件中加入金额阈值，小额采购跳过总经理环节

### 3.2 费用报销

**流程**：草稿 → 直属上级审批 → 财务审核 → 出纳付款 → 已完成

- 绑定文档类型：`Expense Claim`
- 财务审核状态可限制只有"财务角色"可操作
- 出纳付款后自动触发会计凭证生成

### 3.3 请假审批

**流程**：草稿 → HR 初审 → 部门经理审批 → 已批准

- 绑定文档类型：`Leave Application`
- 可加条件：年假余额不足时自动拒绝（在转换条件中调用 `get_leave_balance()`）

### 3.4 合同审批

**流程**：草稿 → 法务审核 → 商务负责人 → 总经理签批 → 归档

- 绑定自定义文档类型（需先通过 DocType 创建合同模块）
- 归档状态可触发自动上传至文件管理系统

## 四、邮件通知配置

在工作流状态中勾选"发送邮件提醒"后，还需配置**通知规则**：

`设置 → 通知 → 新建通知`

| 配置项 | 说明 |
|--------|------|
| 文档类型 | 与工作流绑定的单据类型 |
| 触发条件 | 选择"工作流状态变更" |
| 收件人 | 支持角色、字段引用（如 `doc.approver`）、固定邮箱 |
| 抄送 | 填写抄送邮箱，多个用逗号分隔 |
| 邮件模板 | 使用 Jinja2 语法，可引用 `doc` 对象中的任意字段 |

**拒绝通知示例模板**：

```jinja
您好 {{ doc.employee_name }}，

您提交的 {{ doc.name }} 已被拒绝。
拒绝原因：{{ doc.workflow_state_reason }}

如有疑问请联系您的直属上级。
```

> **截图说明**：通知配置页面的"收件人"区域支持动态字段，点击"+"可添加多条收件规则，分别对应审批通过和拒绝两种场景。

## 五、移动端审批：企业微信 / 钉钉集成

### 企业微信

通过 ERPNext 的 Webhook 功能，在工作流状态变更时推送消息到企业微信群机器人：

1. `设置 → Webhook → 新建`，触发事件选择 `on_update`
2. 条件过滤：`doc.workflow_state == "Pending Approval"`
3. 请求 URL 填写企业微信机器人 Webhook 地址
4. 请求体使用 JSON 格式，引用 `doc` 字段拼装消息内容

### 钉钉

流程与企业微信类似，钉钉机器人 Webhook 支持 Markdown 格式消息，可在消息中嵌入单据直链，审批人点击即可跳转到 ERPNext 移动端完成审批。

> 建议配合 ERPNext 的 PWA 模式使用，在手机浏览器中添加到主屏幕，体验接近原生 App。

## 六、审批记录与合规审计

ERPNext 对所有工作流操作自动记录在**文档时间线（Timeline）**中，包含：

- 操作人、操作时间、状态变更前后值
- 审批意见（可在转换动作中配置"需要填写备注"）
- 系统自动生成的版本快照（通过 `版本` 标签页查看）

**合规审计建议**：

1. 开启 `设置 → 系统设置 → 启用文档版本控制`，保留完整修改历史
2. 定期导出审批日志：通过"报表 → 工作流操作日志"生成 Excel 报表
3. 对敏感单据（合同、大额采购）设置"已批准后禁止修改"，在工作流状态中将"允许编辑"设为否

> **截图说明**：单据底部的"活动"面板会按时间倒序展示所有审批动作，每条记录包含操作人头像、时间戳和状态变更说明，可直接截图作为审计凭证。

## 小结

| 场景 | 推荐方案 |
|------|----------|
| 简单两级审批 | 原生工作流 + 邮件通知 |
| 金额条件路由 | 转换条件 Python 表达式 |
| 移动端推送 | Webhook + 企业微信/钉钉机器人 |
| 合规留痕 | 文档版本控制 + 审批日志报表 |

工作流配置完成后，建议在测试环境走完整流程验证每个状态转换，再上线生产环境。
