# ERPNext 一人公司避坑指南：如何一键重置及配置“会计科目表”

## 痛点切入 (一人公司遇到的坑)

对于初创企业，尤其是那些目标申请“高新技术企业”认证的小微公司，在初始化 ERPNext 财务模块时，最头疼的莫过于“会计科目表 (Chart of Accounts)” 的配置。

很多第一次接触 ERPNext 的朋友（包括我），在按照中国标准的财务要求添加 87 个以上的科目时，都是在网页上一条一条手动敲进去的。但在这条“手工录入”的路上，往往会踩到以下几个巨坑：

1. **父科目依赖问题**：好不容易整理了一个 CSV 文件准备通过“数据导入工具 (Data Import)”导入，却报错“父目录不存在”。这是因为 ERPNext 的科目表是树形结构，子科目的创建必须建立在父科目已经存在的基础上。
2. **科目绑定导致无法删除**：有时手工输入填错了分类，想要删掉重来，却收到报错：“该科目在公司 (Company) 页面已绑定，无法删除！”。当你找到公司设置页面，发现除了默认的银行账户外，还有库存设置、销售设置等好多个地方都绑定了这些默认科目。因为系统根据 `Account Type` 做了强制筛选，如果你想更改或删除，只能先手动去各个设置页面把选项清空。
3. **“牵一发而动全身”的麻烦**：由于公司的各个模块都已经绑定了旧的科目，如果我们要彻底重构科目表结构，这近百个科目的增删改查将会演变成一场噩梦。

## 解决方案 (Frappe 的原生解法)

作为遵循 **Frappe First** 的全栈开发者，既然底层数据结构是受严格校验的，我们最好的做法不是修改底层代码屏蔽校验，而是编写一个 **Python 自动化脚本**。

通过调用 ERPNext 的原生的 REST API (`/api/resource/Account` 和 `/api/resource/Company`)，我们可以完成一套标准的**“解绑-清理-重建-重绑”**工作流：

1. **自动解绑**：利用 API 向公司 (Company) 的各个默认科目字段发送空值请求。
2. **自下而上清理**：先筛选出叶子科目（`is_group = 0`）进行删除，然后再循环删除父级科目（`is_group = 1`），避免树结构锁死的报错。
3. **按序一键重建**：通过读取事先编排好顺序的 CSV 文件（确保父科目在子科目之前），重新向 ERPNext 发送创建科目的请求。
4. **自动回填绑定**：最后再将新建好的“默认银行存款”、“应收账款”等关键科目写回公司设置中。

我们已经编写了这样一个开箱即用的自动化工具，只需要执行 `python sync_chart_of_accounts.py`，几秒钟就能帮你完成科目表的一键重置。

## 避坑指南 (配置时的注意事项)

在使用自动化脚本一键重置科目表时，请务必注意以下几点：

- **数据隔离**：确保你的 CSV 文件中，每个科目的名称后缀都带上了公司名称（例如：`银行存款 - 青皮（上海）信息科技有限公司`）。因为在 ERPNext 多公司的环境中，账户名称必须是全局唯一的。
- **不可撤销的删除**：自动化脚本会强行删除所有的科目。**如果你的系统中已经存在任何交易记录（Transaction，例如凭证、发票、入库单等），系统会自动拦截并拒绝删除被引用的科目。** 因此，这个方案**仅适用于系统初始化或新公司刚开账时使用**。如果有业务数据，必须先撤销 (Cancel) 并删除 (Delete) 相应的业务单据。
- **CSV 的排序逻辑**：在准备 CSV 导入模板时，请严格按照根节点（资产/负债/权益/收入/费用）、一级分类、二级科目的顺序从上往下排列。
- **复式记账法原则**：无论你在前期怎么折腾科目结构，在开始记第一笔账之前，务必通过 `Journal Entry` (日记账凭证) 录入期初余额 (Opening Balance)，并且必须保证“借贷必相等”。
