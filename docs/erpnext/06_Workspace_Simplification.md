# 06 工作区简化与权限配置指南

## 痛点切入
ERPNext 默认的工作区（Workspace）布局和权限体系虽然功能强大，但对于小微团队或特定业务场景来说，往往显得过于复杂。新用户登录后面对数十个模块和数百个菜单项，容易产生“功能恐惧症”。同时，默认的权限设置可能不符合中国企业的实际管理需求，比如财务人员需要看到所有会计科目但不应修改系统设置。

## 我们的简化原则
1. **按角色定制**：为不同岗位（财务、销售、采购、管理员）创建专属工作区
2. **隐藏非必要功能**：移除与当前业务无关的模块和菜单
3. **权限最小化**：遵循“最小必要权限”原则，避免权限滥用
4. **渐进式启用**：随着业务发展逐步开放更多功能，而非一次性全开

## 一、工作区（Workspace）定制化

### 1.1 创建角色专属工作区
ERPNext v16 引入了更灵活的工作区系统，我们可以为不同角色创建专属视图。

#### 示例：为财务人员创建简化工作区
1. 进入 **设置 (Settings)** → **工作区 (Workspace)**
2. 点击 **新建工作区**
3. 填写基本信息：
   - **标题**: `财务中心`
   - **图标**: `accounting`
   - **公共**: 勾选（让所有用户可见）
   - **限制用户**: 选择财务相关角色

4. 配置卡片内容：
   ```
   卡片1: 总览
     - 快捷链接: 仪表板、待办事项、日历
     - 图表: 月度收入趋势、应收账款账龄
   
   卡片2: 日常操作
     - 快捷链接: 销售发票、采购发票、付款记录
     - 最近文档: 最近创建的发票
   
   卡片3: 报表中心
     - 快捷链接: 损益表、资产负债表、现金流量表
     - 快捷过滤器: 本月、本季度、本年
   ```

### 1.2 隐藏非必要模块
对于小微团队，可以隐藏以下不常用模块：

```python
# 在 Site Config 中配置隐藏的模块
# 路径: 设置 → 网站设置 → 网站配置
{
  "hide_modules": [
    "Manufacturing",      # 生产制造（如无生产业务）
    "Projects",           # 项目管理（如使用独立工具）
    "Quality",            # 质量管理
    "Support",            # 客服支持（如使用独立系统）
    "Education"           # 教育培训
  ]
}
```

### 1.3 自定义主页仪表板
为不同角色设置不同的默认主页：

1. **管理员**: 系统监控 + 用户活动
2. **财务**: 财务指标 + 待处理单据
3. **销售**: 销售漏斗 + 客户跟进
4. **采购**: 供应商管理 + 库存预警

**设置方法**:
- 进入 **设置** → **用户**
- 编辑用户 → **角色和权限** 标签页
- 在 **默认工作区** 中选择对应的定制工作区

## 二、权限体系优化

### 2.1 角色权限矩阵
建议为小微团队创建以下核心角色：

| 角色 | 描述 | 关键权限 |
|------|------|----------|
| **系统管理员** | 全权管理 | 所有权限（User Manager, System Manager） |
| **财务主管** | 财务全流程 | 会计、开票、付款、报表（Accounts Manager） |
| **销售经理** | 销售业务 | 客户、销售订单、发票（Sales Manager） |
| **采购专员** | 采购业务 | 供应商、采购订单、收货（Purchase User） |
| **库存管理员** | 仓库管理 | 物品、仓库、库存调整（Stock User） |
| **普通员工** | 基础操作 | 个人资料、请假申请、报销提交（Employee） |

### 2.2 创建自定义角色
对于特殊需求，可以创建自定义角色：

1. 进入 **设置** → **角色和权限** → **角色**
2. 点击 **新建角色**
3. 示例：`财务只读` 角色
   - **角色名称**: `财务只读`
   - **权限**:
     - 会计: 仅查看
     - 发票: 仅查看  
     - 付款: 仅查看
     - 报表: 所有报表可查看
     - **禁止**: 创建、修改、删除、提交、取消

### 2.3 文档级权限控制
ERPNext 支持精细的文档级权限控制：

#### 示例：限制销售员只能看到自己的客户
```python
# 在客户 (Customer) DocType 的权限规则中设置
{
  "role": "Sales User",
  "permlevel": 0,
  "match": "owner",
  "user_match": ["owner"]
}
```

#### 示例：部门数据隔离
```python
# 设置部门级数据可见性
{
  "role": "Department Manager",
  "permlevel": 0,
  "match": "department",
  "user_match": ["department"]
}
```

## 三、界面简化配置

### 3.1 简化表单字段
对于复杂表单，可以隐藏非必要字段：

```python
# 在自定义脚本中配置字段可见性
frappe.ui.form.on('Sales Order', {
  refresh: function(frm) {
    // 对销售员角色隐藏技术性字段
    if (frappe.user.has_role('Sales User')) {
      frm.set_df_property('taxes_and_charges', 'hidden', 1);
      frm.set_df_property('payment_terms', 'hidden', 1);
      frm.set_df_property('shipping_rule', 'hidden', 1);
    }
  }
});
```

### 3.2 自定义列表视图
简化列表视图，只显示关键信息：

1. 进入任意列表页面（如客户列表）
2. 点击右上角 **设置** 图标
3. 选择 **列表设置**
4. 调整显示的列，建议保留：
   - 客户: 名称、客户组、电话、邮箱
   - 销售订单: 客户、总计、状态、创建日期

### 3.3 批量操作简化
禁用不必要的批量操作，减少误操作风险：

```python
# 在网站配置中禁用批量删除
{
  "disable_bulk_delete": 1,
  "disable_bulk_edit": 0  # 保留批量编辑
}
```

## 五、最佳实践建议

### 5.1 权限审计定期检查
建议每月执行一次权限审计：
1. 检查是否有离职员工账号未禁用
2. 审查权限异常（如普通员工有管理员权限）
3. 验证数据隔离是否生效

### 5.2 渐进式权限开放
不要一次性给予所有权限：
- **第1个月**: 仅基础查看权限
- **第2个月**: 增加创建权限
- **第3个月**: 增加修改权限
- **第6个月**: 考虑给予删除权限（如必要）

### 5.3 权限变更记录
所有权限变更应有记录：
- 谁申请的权限
- 为什么需要这个权限
- 谁批准的
- 何时生效/失效

### 5.4 紧急权限回收机制
建立紧急情况下快速回收权限的流程：
1. 立即禁用用户账号
2. 导出该用户的所有操作日志
3. 审查是否有异常操作
4. 根据审查结果决定是否恢复部分权限

## 六、常见问题解答

### Q1: 如何快速重置某个用户的所有权限？
A: 进入 **设置 → 用户**，编辑用户，在 **角色** 标签页移除所有角色，然后重新分配基础角色。

### Q2: 权限配置错了如何恢复？
A: ERPNext 有权限回滚机制，可以进入 **设置 → 日志** 查看权限变更记录，或使用 `bench --site site.local restore-permissions` 命令恢复备份。

### Q3: 自定义角色会影响系统升级吗？
A: 自定义角色和权限配置通常不会受系统升级影响，但建议在升级前导出权限配置作为备份。

### Q4: 如何测试权限配置是否生效？
A: 创建测试用户，分配对应角色，然后登录测试账号验证各项功能是否符合预期。

## 七、下一步优化方向

1. **基于部门的动态权限**：根据用户所属部门动态调整数据可见范围
2. **时间限制权限**：为临时权限设置有效期，到期自动回收
3. **审批流集成**：权限申请与审批流程集成
4. **权限分析报表**：生成权限使用情况分析报表，优化权限分配

通过合理的工作区简化和权限配置，可以让 ERPNext 更好地适应小微团队的实际需求，降低学习成本，提高使用效率，同时确保数据安全。
