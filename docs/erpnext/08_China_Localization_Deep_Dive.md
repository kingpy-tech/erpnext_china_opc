# ERPNext 中国本土化深度指南：财务合规、税务处理与业务流程适配

> 本文面向已完成基础安装、准备在中国企业环境中真正跑通 ERPNext 的实施者。不讲虚的，直接说清楚几个最容易卡住的本土化难点。

---

## 一、增值税发票处理

中国增值税体系分为**增值税专用发票**（可抵扣）和**增值税普通发票**（不可抵扣），这是 ERPNext 默认税务模型里没有的概念，需要手动适配。

**核心配置思路：**

1. 在「税务模板」里分别建两套模板：`增值税专票 13%`、`增值税普票 13%`，以及小规模纳税人适用的 `增值税 3%`。
2. 在销售发票和采购发票的自定义字段里加入「发票类型」（专票/普票）、「发票代码」、「发票号码」、「校验码」字段，方便后续与金税系统核对。
3. 进项税抵扣：采购专票认证后才能抵扣，建议在采购发票里加「认证状态」字段，未认证的不计入可抵扣进项。
4. 税率切换：餐饮、交通等服务类采购税率不同，建议按供应商类型预设默认税务模板，减少手工选择出错。

**一个常见坑：** ERPNext 默认的 Tax Account 是含税价倒推，而中国发票是价税分离的。务必在「公司设置」里确认「含税价格」选项与你的开票习惯一致，否则账面金额会对不上。

## 二、中国财务报表：资产负债表与利润表

ERPNext 自带的财务报表是按 IFRS/GAAP 逻辑设计的，中国企业会计准则（CAS）的科目结构和报表格式有差异。

**适配方法：**

1. **科目表导入**：参考本项目 `03_Chart_of_Accounts_Import.md`，导入符合中国会计准则的科目体系（资产类 1xxx、负债类 2xxx、所有者权益 3xxx、成本类 4xxx、损益类 5xxx/6xxx）。
2. **自定义报表**：ERPNext 的「财务报表」支持自定义分组，按中国准则重新映射科目到报表行项目。资产负债表需要区分流动/非流动，利润表需要体现「营业收入 → 营业利润 → 利润总额 → 净利润」的层级。
3. **期间调整**：中国会计年度为自然年，ERPNext 默认支持，但要注意「会计期间」设置里的期末结转逻辑，避免跨年数据错位。

**实用建议：** 先用 ERPNext 的「试算平衡表」验证借贷平衡，再导出数据到 Excel 按中国格式整理，等自定义报表调好之后再全面切换。

## 三、社保公积金计算

这是中国 HR 模块里最复杂的部分，因为各城市缴费基数和比例不同，而且每年 7 月会调整。

**配置框架：**

1. 在「薪资组成」里建立以下扣款项：
   - 养老保险（个人 8%）
   - 医疗保险（个人 2% + 大病统筹 3 元）
   - 失业保险（个人 0.5%）
   - 住房公积金（个人比例按城市，上海一般 7%）
2. 缴费基数上下限每年更新，建议用「薪资变量」存储，方便年度批量调整。
3. 企业部分（养老 16%、医疗 10% 等）计入人工成本，需要在薪资条目里单独列出，方便财务做成本分摊。

**个税联动：** 社保公积金个人部分可以税前扣除，ERPNext 的薪资模块支持在个税计算前先扣除这部分，配置时注意计算顺序。

## 四、人民币大写

中国财务凭证和支票上必须填写人民币大写金额（壹贰叁肆伍陆柒捌玖拾佰仟万亿），ERPNext 默认没有这个功能。

**解决方案：**

1. 安装社区插件 `erpnext_china`（frappe 生态），其中包含人民币大写转换函数。
2. 或者在打印格式（Print Format）里用 Jinja 模板调用自定义 Python 函数实现转换，函数逻辑约 30 行，网上有现成实现。
3. 在「付款凭证」和「销售发票」的打印模板里引用该函数，确保打印输出符合财务规范。

## 五、与金税系统对接思路

金税系统（百旺、航天信息等）是中国开具增值税发票的法定渠道，ERPNext 本身不直接对接，需要通过中间层实现数据流转。

**主流方案：**

1. **手工导出导入**：从 ERPNext 导出待开票数据（Excel），导入金税盘开票，再将发票信息回填 ERPNext。适合开票量小的企业，零开发成本。
2. **API 中间件**：部分金税服务商（如百旺云、诺诺网）提供 REST API，可以开发一个轻量中间件，监听 ERPNext 的销售发票提交事件，自动推送到金税 API 开票，回调写入发票号码。
3. **frappe-nest 集成**：青皮科技正在探索通过 frappe-nest 统一管理 ERPNext 与外部系统的集成，金税对接是其中一个场景，后续会有专项文档。

**注意事项：** 金税系统对发票数据格式有严格要求，商品名称、税收分类编码（税收编码）必须与金税系统一致，建议在 ERPNext 的「物料」主数据里增加「税收分类编码」字段，开票时自动带入。

---

## 小结

中国本土化不是一个插件装上去就搞定的事，它是一系列配置、自定义字段、打印模板和外部系统对接的组合。建议按以下顺序推进：

1. 先跑通科目表和基础财务流程
2. 再配置增值税模板和发票字段
3. 然后上 HR 模块和社保计算
4. 最后处理金税对接和打印格式

每一步都先在沙盒环境验证（参考 `02_Advanced_Backup_Restore_Tenant.md`），确认无误再上生产。

有问题欢迎在 GitHub Issues 里提，或者加入我们的社区讨论。
