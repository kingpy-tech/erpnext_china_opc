# ERPNext 多公司与多站点管理实战

本文介绍如何在 ERPNext 中配置多公司架构与多站点部署，适用于集团企业、控股公司及需要独立核算的业务场景。

---

## 一、多公司适用场景

| 场景 | 说明 |
|------|------|
| 集团管控 | 母公司统一管理多个子公司，合并出集团财务报表 |
| 子公司独立核算 | 各子公司拥有独立科目表、独立账期、独立报表 |
| 共享服务中心 | HR、IT、财务等职能集中在一个公司实体，向其他公司收费 |

ERPNext 原生支持在同一个站点（Site）内创建多个公司实体，共享用户、物料主数据，同时保持财务数据隔离。

---

## 二、多公司配置

### 2.1 创建公司树

在 **设置 → 公司** 中新建子公司，并在 `母公司` 字段中选择上级公司，即可形成树形结构：

```
青皮科技集团（母公司）
├── 青皮上海（子公司）
└── 青皮北京（子公司）
```

每个公司需独立配置：

- 默认货币与会计科目表
- 财年起止月份
- 默认银行账户与成本中心

### 2.2 内部交易配置

公司间交易需在 **设置 → 公司间交易设置** 中启用，并为每对公司配置：

```yaml
# 示例：上海 → 北京 内部采购
发起公司: 青皮上海
目标公司: 青皮北京
应付科目 (上海): 2202 - 应付账款
应收科目 (北京): 1122 - 应收账款
```

创建公司间采购订单时，系统会自动在目标公司生成对应的销售订单。

### 2.3 合并报表

在 **会计 → 合并财务报表** 中选择母公司，系统将汇总所有子公司数据并消除内部往来，生成集团级资产负债表与利润表。

> ⚠️ 合并报表要求所有子公司使用相同的基础货币，或已正确配置汇率。

## 三、多站点架构

当不同公司需要完全隔离的数据库（如数据安全合规要求），应采用多站点部署。

### 3.1 单服务器多站点

Frappe/ERPNext 天然支持一台服务器运行多个 Site，每个 Site 拥有独立的 MariaDB 数据库：

```bash
# 创建新站点
bench new-site company-b.example.com \
  --mariadb-root-password <pwd> \
  --admin-password <pwd>

# 为新站点安装 ERPNext
bench --site company-b.example.com install-app erpnext

# 设置多站点 Nginx
bench setup nginx
sudo service nginx reload
```

### 3.2 独立数据库与资源隔离

| 资源 | 隔离方式 |
|------|----------|
| 数据库 | 每个 Site 对应独立 MariaDB 数据库 |
| 文件存储 | `sites/<site-name>/private/` 与 `public/` 目录独立 |
| 定时任务 | Scheduler 按 Site 独立运行 |
| Redis 缓存 | 默认共享，可通过 `common_site_config.json` 指定独立实例 |

## 四、权限管理

### 4.1 跨公司权限控制

ERPNext 通过 **用户权限（User Permission）** 实现数据隔离：

```
设置 → 用户权限 → 新建
允许: 公司
值: 青皮上海
用户: zhang@example.com
```

该配置使 `zhang@example.com` 只能查看和操作"青皮上海"的单据。

### 4.2 角色继承与数据隔离

- 角色（Role）在系统级定义，所有公司共享
- 用户权限（User Permission）在公司级过滤，实现数据隔离
- 推荐为每个子公司创建专属角色（如 `上海财务`），再叠加用户权限，形成双重隔离

## 五、公司间内部交易流程

### 5.1 采购与销售联动

1. 在 A 公司创建 **公司间采购订单**，选择供应商为 B 公司
2. 系统自动在 B 公司生成 **销售订单**
3. B 公司确认后，双方分别完成收货/发货与发票核销

### 5.2 资金调拨

公司间资金划转通过 **日记账（Journal Entry）** 完成：

```
借：青皮北京 - 内部往来（青皮上海）  100,000
贷：青皮上海 - 内部往来（青皮北京）  100,000
```

在合并报表时，上述内部往来科目将被自动消除。

---

## 六、合并报表与内部消除

### 6.1 集团财务报表生成步骤

1. 进入 **会计 → 合并财务报表**
2. 选择 `母公司` 与 `财年`
3. 勾选需要合并的子公司
4. 点击 **生成报表**

### 6.2 内部交易消除

系统自动识别并消除以下科目的内部余额：

- 内部应收 / 内部应付
- 内部销售收入 / 内部采购成本
- 公司间投资与资本

> 💡 建议为内部往来单独设立科目段（如 `29xx` 系列），便于合并时精准消除，也方便日常对账。

---

## 小结

- **同一业务实体、数据共享**：使用单站点多公司，配置公司树与用户权限
- **数据完全隔离、合规要求高**：使用多站点部署，每站点独立数据库
- 内部交易与合并报表是多公司架构的核心，建议在上线前完整测试消除逻辑
