# 00 ERPNext 准备篇：为什么选择 1Panel 及其安装指南

## 痛点切入

在部署 ERPNext 时，传统的方式是使用 Frappe 官方提供的 `bench install` 脚本直接在服务器的系统环境中安装。但很多小微团队或独立开发者在尝试这种方式时，常常会被各种依赖问题（Python 版本、Node.js 版本、Redis、MariaDB 等）折磨得痛苦不堪。

虽然使用纯 Docker Compose 部署能解决环境隔离的问题，但对于非专业的运维人员来说，日常的容器管理、日志查看、反向代理配置以及 SSL 证书申请，仍然需要敲击大量繁琐的命令行。这极大地增加了搭建属于自己的数字化后台的门槛。

## 我们的折腾解法：为什么选择 1Panel

为了降低运维门槛，我们强烈推荐在开始部署 ERPNext 之前，先在服务器上安装 **1Panel** —— 一个现代化、开源的 Linux 面板。

选择 1Panel 有以下几个核心优势：
1. **自带容器编排（Docker Compose）可视化界面**：你不需要在命令行手写 `docker-compose up -d`，直接在 Web 界面粘贴配置，一键就能拉起复杂的 ERPNext 集群。
2. **极简的容器管理**：重启服务、查看每个容器（比如 `db`、`backend`、`frontend`）的实时日志，只需点点鼠标即可完成，对于排查 ERPNext 启动报错非常直观。
3. **开箱即用的反向代理与 HTTPS**：当你需要对外提供服务（例如使用 `erp.qingpy.cn`）时，1Panel 内置的 OpenResty 网站管理模块和自动化的 Let's Encrypt 证书申请，能让你在几分钟内为 ERPNext 开启安全的 HTTPS 访问。
4. **轻量且安全**：使用 Go 语言编写，占用资源极低（百兆内存即可运行），非常适合微型服务器或云服务器，并且提供面板端口自定、安全入口等完善的防护机制。

## 1Panel 安装指南（以 Ubuntu 为例）

如果你使用的是腾讯云、阿里云的轻量应用服务器或 CVM，推荐选择纯净的 **Ubuntu 24.04 LTS** 操作系统。

### 1. 登录服务器
使用 SSH 客户端（如 macOS/Linux 自带的终端，或 Windows 的 Termius、Xshell 等）连接到你的服务器：
```bash
ssh root（或其他用户名）@你的服务器IP
```

### 2. 执行一键安装脚本
进入服务器后，直接复制并执行 1Panel 官方提供的一键安装脚本：

```bash
curl -sSL https://resource.fit2cloud.com/1panel/package/quick_start.sh -o quick_start.sh && sudo bash quick_start.sh
```

### 3. 安装过程配置
脚本运行后会自动下载安装包并开始安装，期间会提示你进行一些基础设置：

1. **设置 1Panel 端口**：默认是随机生成的一个高位端口（如 `18888`），你可以直接回车使用默认，或者输入自定义端口。
2. **设置安全入口**：默认会生成一串随机字符串（如 `/a1b2c3d4`），这是为了防止面板被恶意扫描。
3. **设置管理员账号与密码**：系统会要求你输入登录 1Panel 面板的用户名和密码。

> **⚠️ 注意（云服务器必看）**：
> 如果你使用的是腾讯云或阿里云服务器，必须登录云服务商的控制台，在 **安全组**（或防火墙）规则中，手动添加一条入站规则：
> - **协议/端口**：TCP / 你刚刚设置的 1Panel 端口（如 18888）
> - **源地址**：`0.0.0.0/0`（允许所有 IP 访问，或者限制为你家里的 IP）
>
> 否则，你将无法在浏览器中打开 1Panel。

### 4. 登录并初始化 1Panel
安装完成后，终端会打印出类似下面的访问地址：
```text
面板访问地址：http://服务器IP:端口/安全入口
```

1. 复制该地址到浏览器打开。
2. 输入你刚才设置的用户名和密码。
3. 登录成功后，你会看到 1Panel 的工作台。如果系统提示是否安装 Docker 等依赖软件，请一律选择 **安装**（1Panel 会全自动帮你配置好 Docker 环境，这也是我们部署 ERPNext 所必需的）。
