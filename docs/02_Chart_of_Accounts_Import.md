# 02 高新技术企业会计科目表集成指南

## 痛点切入
ERPNext 原生内置的中国区会计科目表比较基础。对于中国的小微企业，尤其是目标为申请“高新技术企业”认证的公司，对研发费用的核算（如：资本化支出与费用化支出、人员人工、直接投入等）有非常严格的要求。如何在初期建立规范的科目表，是财务合规的第一步。

## 解决方案
通过 ERPNext 自带的 **Data Import (数据导入)** 功能，将预先配置好的、针对高新企业优化的 `erpnext_accounts_backup.csv` 直接导入到系统中。这是一种完全无代码 (No-Code) 的配置方式，完全符合 Frappe 的最佳实践。

## 配置步骤

### 1. 准备科目表导入文件
在 `config_package/chart_of_accounts/` 目录下，我们已经准备好了定制的 `erpnext_accounts_backup.csv`，其中包括：
- 一级科目（资产、负债、所有者权益、成本、收入、费用）。
- 针对高新企业的研发支出明细科目（4301 系列）。
- 带有公司缩写 (QP) 的科目名称隔离设计。

### 2. 进入数据导入工具
1. 登录 ERPNext，在顶部全局搜索栏输入 `Data Import`（数据导入），并点击新建。
2. **DocType (文档类型)** 选择 `Account` (科目)。
3. **Action (操作)** 选择 `Insert New Records` (插入新记录)。如果是更新系统自带的部分科目，可选择 `Update Records`。
4. 保存后，点击 **Attach File** 上传我们的 `erpnext_accounts_backup.csv` 文件。

### 3. 执行导入与校验
1. 上传文件后，系统会自动映射 CSV 表头和系统字段（如：Account Name, Parent Account, Is Group, Account Type 等）。
2. 点击右上角的 **Start Import (开始导入)** 开始执行。
3. 导入完成后，进入 **Chart of Accounts (会计科目表)** 树状视图，展开各个层级，检查“研发支出”及其子科目是否正确挂载在成本类科目下。

## 避坑指南
- **父子节点依赖**：CSV 导入时，父科目必须比子科目先被创建。我们提供的 CSV 已经按照依赖关系调整好了行顺序，请勿随意打乱电子表格的行顺序，否则会报 Parent Not Found 错误。
- **Company 字段匹配**：CSV 文件中的 `Company` 字段当前值为“青皮（上海）信息科技有限公司”。如果你在 ERPNext 中创建的公司名称不同，请在导入前全局替换 CSV 文件里的公司名称，否则导入时会校验失败。
- **Root Type 绑定**：顶级节点（如资产、负债、权益、收入、费用）必须有合法的 `Root Type`，否则后期的资产负债表与利润表将无法正常生成。
- **借贷必相等**：在初始化设置时，确保设置的 Opening Balance (期初余额) 借贷双方总额相等。
